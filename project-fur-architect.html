Unity C# & Compute Shaders

Fur Architect

Genre: 3D Grooming & Haircut Simulator.
Role: Project Architect & Gameplay Developer
A technical achievement in GPU optimization, featuring a real-time grooming engine where every cut and color is calculated with mathematical precision.

Core Gameplay Arc

Players act as a stylist using high-tech tools to match a requested "blueprint" look. The experience blends creative expression with technical puzzle-solving.

  • GPU Grooming Engine: Leveraging Custom Render Textures to handle real-time hair shaving, regrowing (serum), and dyeing without CPU overhead.
  • Compute Shader Scoring: Implemented a high-performance verification engine that compares the current fur state against the goal mask in real-time to provide 1-3 star ratings.
  • Diegetic UI: Enhanced immersion by removing floating menus. All feedback is provided through an in-world TV monitor and diegetic sticky notes.
  • Event-Driven Workflow: Optimized code maintainability using a centralized EventManager, allowing for decoupled interaction between the character, tools, and UI.

Technical Deep Dive

The Scoring Engine

By moving the haircut verification logic to a Compute Shader, I was able to perform pixel-by-pixel comparisons on 1024x1024 textures in less than 0.1ms per frame, ensuring perfectly smooth performance even on lower-end hardware.

Diegetic UX

The UI is integrated directly into the 3D world. The stylist's TV displays the final score and goal reference, while tools are physically grabbed from the workbench, maintaining a high level of tactile feedback.
